# S-EMG-main 功能梳理

## 项目定位

这个仓库的核心是一个基于表面肌电信号（sEMG）的手势识别原型，完成了从硬件采集到模型训练，再到实时识别的完整闭环。

适合按模块迁移到其他工程中。

---

## 主要功能

项目整体实现了以下 7 个功能：

1. Arduino 端采集 3 路肌电信号，并做基础滤波后通过串口输出
2. Python 端按手势类别分批采集训练数据
3. 将连续时序信号切分成固定长度窗口
4. 对每个窗口提取时域、频域、AR 和小波包特征
5. 使用随机森林训练分类模型
6. 输出训练集和测试集精度
7. 持续读取实时串口数据并进行在线手势预测

---

## 模块功能拆解

## 1. 采集端功能

Arduino 端负责：

1. 从 `A0/A1/A2` 读取 3 路模拟肌电信号
2. 使用 `EMGFilters` 做滤波处理
3. 通过串口按 CSV 格式输出每一帧数据

串口输出格式为：

```text
ch0,ch1,ch2
```

关键参数：

- 串口波特率：`115200`
- 蓝牙串口波特率：`9600`
- Arduino 端设置的采样率：`500Hz`

---

## 2. Python 串口读取功能

Python 端通过 `pyserial` 从串口读取数据，并将每一行解析为 3 个浮点数。

输入假设：

1. 每行数据都能正常 `decode`
2. 每行以逗号分隔
3. 每行恰好对应 3 路通道值

旧版直接在脚本中写读取逻辑，重构版封装成了：

- `readData(count, threshold=1000)`

这个函数的职责是：

1. 连续读串口
2. 过滤异常值
3. 收集指定数量的样本点

---

## 3. 训练数据采集功能

项目默认采集 4 类手势：

- `front`
- `back`
- `left`
- `right`

重构版中由以下函数完成：

- `collectData(gestureGroup=['front','back','left','right'], samples_per_gesture=2000, gestures_repeat=4)`

作用：

1. 逐个提示用户做手势
2. 每种手势采集固定数量样本
3. 支持多轮重复采集
4. 将数据按手势类别保存到字典中

数据结构大致为：

```python
{
    "front": [...],
    "back": [...],
    "left": [...],
    "right": [...]
}
```

---

## 4. 数据保存功能

重构版提供：

- `saveDataToExcel(data)`

作用：

1. 在 `data/` 目录下创建带时间戳的 Excel 文件
2. 每个手势类别写入一个 sheet

这部分适合迁移时保留为“数据记录模块”。

---

## 5. 标签化与标准化组织

重构版中的：

- `normalizeData(data)`

作用：

1. 将不同手势的数据拼接成统一的 `emg` 数组
2. 为不同手势生成数值标签

输出：

- `emg`：形如 `N x 3` 的肌电数据
- `label`：长度为 `N` 的类别标签数组

---

## 6. 滑窗切片功能

重构版中的：

- `convertDataToImageData(emg, label, timeWindow=200, strideWindow=200)`

作用：

1. 按类别分别取出连续信号
2. 将连续信号切成固定长度窗口
3. 为每个窗口生成对应标签

当前默认参数：

- 窗口长度：`200`
- 步长：`200`

这意味着当前实现本质上是“非重叠窗口”切分。

输出：

- `imageData`：形如 `样本数 x 200 x 3`
- `imageLabel`：每个窗口对应的类别标签

---

## 7. 特征提取功能

项目的核心算法集中在 `feature_utils.py` 中。

已实现的特征包括：

### 时域特征

1. `featureRMS`
   - 均方根
2. `featureMAV`
   - 平均绝对值
3. `featureWL`
   - 波形长度
4. `featureZC`
   - 过零率
5. `featureSSC`
   - 斜率符号变化次数
6. `featureVAR`
   - 方差
7. `featureWA`
   - Willison Amplitude
8. `featureMCV`
   - 三阶矩相关特征
9. `featureME`
   - 频谱均值相关特征

### 频域特征

10. `featureMPF`
    - 平均功率频率
11. `featureSM2`
    - 二阶谱矩
12. `featureMF`
    - 中值频率相关实现

### 参数模型特征

13. `featureAR`
    - 7 阶自回归系数

### 时频特征

14. `featureWPT`
    - 小波包分解特征
    - 包含每个频带的能量、均值、方差

---

## 8. 单窗口特征拼接

重构版中的：

- `featureStackForImageData(imageData)`

作用：

1. 对一个 `200 x 3` 的窗口提取所有特征
2. 将所有通道特征横向拼接成一个一维特征向量

这部分是迁移时最值得直接复用的函数之一。

---

## 9. 批量特征提取

重构版中的：

- `featureExtraction(imageData, imageLabel)`

作用：

1. 遍历所有窗口样本
2. 调用 `featureStackForImageData`
3. 生成训练用特征矩阵和标签数组

输出：

- `featureData`
- `featureLabel`

---

## 10. 模型训练功能

项目使用：

- `RandomForestClassifier`

主要流程：

1. 使用 `train_test_split` 划分训练集和测试集
2. 用随机森林拟合训练数据
3. 计算训练精度和测试精度

模型类型：

- 随机森林分类器

用途：

- 对 EMG 窗口特征进行 4 分类手势识别

---

## 11. 模型保存功能

重构版中的：

- `saveModel(rf_model, accuracy)`

作用：

1. 在 `model/` 目录保存训练好的模型
2. 使用 `joblib` 存储
3. 文件名中带有精度和时间戳

这部分迁移时可以直接扩展成模型版本管理。

---

## 12. 实时识别功能

训练完成后，程序进入实时预测循环。

流程：

1. 连续从串口读取 `200` 个点
2. 拼成一个实时窗口
3. 提取相同特征
4. 输入训练好的随机森林
5. 输出预测类别

这部分实现的是“在线推理”能力。

---

## 完整数据流

项目的完整流程可以概括为：

```text
Arduino采样
-> 串口输出三通道EMG
-> Python读取原始信号
-> 按手势采集训练数据
-> 切分为200点窗口
-> 提取多种特征
-> 随机森林训练
-> 实时读取串口窗口
-> 实时预测手势类别
```

## 迁移时需要特别注意的问题

### 1. 采样率不一致

Arduino 端配置的是：

- `500Hz`

但 `feature_utils.py` 中部分频域特征写死了：

- `fs = 2000`

这会影响以下特征的物理意义：

1. `featureMPF`
2. `featureSM2`
3. `featureMF`

迁移时建议先统一采样率定义。

### 2. 串口号硬编码

旧版和重构版分别使用：

- `com6`
- `com3`

迁移时建议改为配置项。

### 3. 实时预测依赖训练流程

当前代码不是“加载已训练模型再预测”，而是：

1. 先现场采集训练数据
2. 现场训练模型
3. 训练完成后立刻进入预测

如果新工程需要部署，建议拆分成：

- 离线训练
- 在线加载模型推理

### 4. 异常处理较弱

项目中大量使用：

```python
except:
    pass
```

迁移时建议补充：

1. 串口超时处理
2. 非法数据日志
3. 数据维度校验
4. 模型输入特征长度校验

### 5. 窗口切片逻辑需要复核

当前函数名里有 `strideWindow`，但实际实现更接近固定分块，不是严格意义的通用滑窗。

如果你的新工程想做：

1. 重叠窗口
2. 连续流式预测
3. 更小延迟识别

建议重写这一层。

## 迁移建议

建议以 `重构后的代码/main.py` 作为迁移蓝本，而不是旧版 `main.py`。

原因：

1. 函数边界更清晰
2. 更容易拆成独立模块
3. 后续更方便加配置、日志和测试

推荐迁移顺序：

1. 先迁移 `feature_utils.py`
2. 再迁移串口读取和窗口切片
3. 再迁移训练流程
4. 最后补在线预测和模型加载

---

## 一句话总结

这个项目本质上是一个“3通道 sEMG 手势识别原型”，核心能力是：

`采集 -> 分段 -> 特征提取 -> 随机森林训练 -> 实时预测`

如果要迁移，优先保留“特征提取模块”和“重构版主流程函数拆分结构”。
